# AGENTS.md

## Обзор проекта

Это небольшой статический веб-проект: дневник тренировок по фридайвингу.

Проект называется **«Дневник фридайвера»**.

Главная цель текущего этапа — простой MVP: цифровая копия бумажного дневника тренировок.

В проекте нет сборки, пакетного менеджера, фреймворка, backend-кода и внешних зависимостей. Приложение запускается прямо в браузере через `index.html`.

Основные файлы:

- `index.html` — структура страницы, форма тренировки, список записей и блок просмотра деталей.
- `style.css` — внешний вид, адаптивная верстка, цвета, кнопки, карточки и стили формы.
- `script.js` — вся логика приложения: загрузка, сохранение, валидация, отрисовка, редактирование, удаление и просмотр записей.

## Как запускать проект

Проект запускается без сервера.

Достаточно открыть файл:

```text
index.html
```

в браузере.

Если нужно проверить изменения, обнови страницу в браузере.

## Поведение приложения

Тренировки хранятся в `localStorage` браузера по ключу:

```text
freediving_trainings
```

Текущая структура одной записи:

```js
{
  id,
  user_id,
  date,
  type,
  coach,
  mood,
  training,
  notes,
  total_volume,
  dives,
  created_at,
  updated_at
}
```

Поле `user_id` сейчас есть в структуре, но на этапе локального MVP оно не используется для регистрации или авторизации. Не добавляй регистрацию и backend, пока пользователь явно не попросит.

Приложение уже умеет:

- создавать запись о тренировке;
- проверять обязательные поля перед сохранением;
- показывать список сохранённых тренировок;
- сортировать тренировки по дате, от новых к старым;
- открывать подробный просмотр тренировки;
- редактировать существующую запись;
- отменять редактирование;
- удалять запись после подтверждения;
- сохранять данные между перезагрузками страницы;
- выводить дату в русском формате `день.месяц.год`;
- безопасно отображать пользовательский текст через `escapeHTML()`.

## Основные поля дневника

## Безопасность вывода данных

Пользовательский текст нельзя вставлять в `innerHTML` напрямую без экранирования.

Для вывода пользовательских полей используй существующую функцию:

```js
escapeHTML()
```

Её нужно применять для таких полей:

- `coach`;
- `training`;
- `notes`;
- `total_volume`;
- `dives`.

Если добавляется новое текстовое поле, его тоже нужно экранировать при выводе.

Особенно важно: строка вроде:

```html
<script>alert(1)</script>
```

должна отображаться как обычный текст, а не выполняться как HTML или JavaScript.

## Совместимость данных

Осторожно меняй структуру сохранённых записей: у пользователя уже могут быть данные в `localStorage`.

Не очищай `localStorage` без прямой команды пользователя.

Не переименовывай существующие поля без логики миграции.

Если добавляешь новое поле:

1. добавь его в форму в `index.html`;
2. считай значение в обработчике отправки формы;
3. сохрани поле в `createTraining()`;
4. обновляй поле при редактировании;
5. безопасно выводи поле, если оно отображается;
6. учитывай старые записи, где этого поля ещё нет.

Если старые записи могут не иметь нового поля, используй безопасный fallback, например `|| 'Не указано'`.

## Безопасность фридайвинга

Приложение — дневник, а не тренер и не медицинский инструмент.

Не добавляй советы, которые могут подталкивать к опасным тренировкам.

Не поощряй:

- одиночные тренировки;
- рекорды любой ценой;
- игнорирование самочувствия;
- тренировки без напарника;
- тренировки без безопасных условий.

Не давай медицинских диагнозов.

Если в интерфейсе или тексте появляются подсказки по тренировкам, формулируй их осторожно: напоминай про напарника, инструктора и безопасные условия.

## Ручная проверка

Автоматических тестов в проекте нет.

После изменений проверь вручную:

1. Открыть `index.html` в браузере.
2. Создать новую тренировку.
3. Убедиться, что она появилась в списке.
4. Открыть подробности тренировки.
5. Отредактировать тренировку.
6. Отменить редактирование.
7. Удалить тренировку.
8. Обновить страницу и убедиться, что сохранённые данные остались.
9. Ввести HTML-подобный текст, например `<script>alert(1)</script>`, и убедиться, что он отображается как текст.
10. Проверить на мобильной ширине экрана.
